# 두쫀쿠 유형 테스트 PRD

## 1. 프로젝트 개요

### 1.1 프로젝트명

두쫀쿠(두바이 쫀득 쿠키) MBTI 유형 테스트

### 1.2 목적

한국에서 유행 중인 두바이 쫀득 쿠키를 주제로, 사용자의 성격 유형(MBTI)에 맞는 두쫀쿠 유형을 매칭해주는 테스트 웹앱

### 1.3 타겟 사용자

- MZ세대 (20-30대)
- 디저트/트렌드에 관심 있는 사용자
- SNS 공유를 즐기는 사용자

### 1.4 일정

- **MVP 배포**: 1주일 내
- **후속 작업**: GA 연동, 결과 이미지 카드 추가

---

## 2. 핵심 기능

### 2.1 테스트 플로우

```
시작 화면 → 질문 (10-12개) → 결과 화면 → 공유
```

### 2.2 질문 시스템

- **질문 수**: 10-12개
- **질문 형식**: 2지선다 또는 4지선다 (MBTI 4가지 축 판별)
  - E/I (외향/내향)
  - S/N (감각/직관)
  - T/F (사고/감정)
  - J/P (판단/인식)
- **질문 톤**: 두쫀쿠/디저트 관련 상황으로 재미있게 구성

### 2.3 결과 시스템

- **유형 개수**: MBTI 16유형 전부 (각각 두쫀쿠 유형과 매칭)
- **결과 페이지 구성**:
  - 유형명 (예: "당신은 클래식 피스타치오 두쫀쿠!")
  - 성격 설명 (해당 MBTI 특성을 두쫀쿠에 빗대어 설명)
  - 궁합 유형 (잘 맞는 유형 / 안 맞는 유형)

### 2.4 공유 기능

- 카카오톡 공유
- 인스타그램 스토리 공유
- X(트위터) 공유
- 링크 복사
- 결과 이미지 저장 (1차에서는 텍스트 기반, 추후 이미지 카드)

## 3. 디자인 요구사항

### 3.1 톤앤매너

- **귀여운/친근한** 느낌
- 부드러운 파스텔 색감
- 둥글둥글한 UI 요소
- 친근한 말투 (반말 or 친근한 존댓말)

### 3.2 컬러 팔레트 (제안)

- 메인: 베이지/크림 계열 (쿠키 연상)
- 포인트: 피스타치오 그린, 초콜릿 브라운
- 배경: 따뜻한 화이트/아이보리

### 3.3 반응형

- 모바일 퍼스트 (주 사용 환경)
- 태블릿/데스크톱 대응

/result/[type]      # 결과 화면 (16개 유형별 동적 라우팅)

## 10. 개발 순서

1. **프로젝트 구조 셋업** - 폴더/파일 구조, 데이터 타입 정의
2. **시작 화면** - 메인 페이지 UI
3. **테스트 페이지** - 질문 UI, 상태 관리, MBTI 계산 로직
4. **결과 페이지** - 동적 라우팅, 결과 표시 UI
5. **공유 기능** - 카톡, 인스타, X, 링크복사, 이미지저장
6. **스타일링 마무리** - 반응형, 톤앤매너 적용
7. **테스트 & 빌드** - lint, build 확인
